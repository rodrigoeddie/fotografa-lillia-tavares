Proposta 5 de 5 — Extrapolação Ousada

Polaroid Wall

Cards estilo polaroid com rotação leve criam sensação de fotografias expostas numa parede — metáfora direta com o trabalho da fotógrafa. A rotação é puramente decorativa, desativada via prefers-reduced-motion, e não afeta o fluxo de tabulação. Risco estético único, totalmente WCAG AA.

Ranking de Acessibilidade — Melhor para Pior

  1. 1
    Proposta 3 — Carrossel com Scroll Snap Acessível

    Melhor cobertura de ARIA para o padrão carrossel (o mais difícil de tornar acessível): role="region" + aria-roledescription="carrossel" + aria-live="polite" no status + botões 44×44px com aria-label descritivo + scroll-snap CSS sem dependência JS para leitura. Resolve o problema mais crítico do design original (Swiper sem ARIA) com a solução mais elegante e resiliente.

  2. 2
    Proposta 1 — Grade Respirada

    Correção mais completa e de menor risco para implementação: todos os 11 problemas identificados corrigidos, ARIA completo em cada card incluindo o mini-slider interno, hierarquia h3→h4 correta, botões com estados loading e vazio demonstrados. Contraste máximo em todos os textos. Ideal para migração incremental sem redesign visual radical.

  3. 3
    Proposta 4 — Editorial Vertical

    Elimina o padrão carrossel por completo, que é a maior fonte de problemas de acessibilidade. Cada ensaio é um item de lista semântico com heading, metadados e link bem dimensionado. Custo: layout linear pode parecer extenso para 5 itens; paginação ou "ver mais" seria necessária em produção.

  4. 4
    Proposta 5 — Polaroid Wall

    Memorável e coerente com o ofício da fotógrafa; contraste e alvos de toque corretos; rotate desligado via prefers-reduced-motion. Perde pontos por não exibir a categoria de forma tão prominente quanto as outras propostas, e o grid denso pode criar dificuldade de scanning para usuários com baixa visão mesmo com contraste correto.

  5. 5
    Proposta 2 — Masonry Editorial

    Visualmente mais impactante mas depende de overlay escuro sobre imagem para garantir contraste — que varia conforme a foto real (imagem muito clara pode comprometer o ratio de 4.5:1 mesmo com gradiente). Em produção exigiria verificação por imagem ou overlay adaptativo via JS. Fica em último por esse risco estrutural, não por falha de design.
