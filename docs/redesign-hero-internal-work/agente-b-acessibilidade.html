Redesign de Hero — Ensaio Interno

Agente B — Acessibilidade & Design Inclusivo (WCAG 2.2 AA)

5 propostas de redesign da seção hero da interna de trabalho de Camila Nogueira (Corporativo). Foco total em WCAG 2.2 AA: contraste ≥ 4.5:1 calculado par a par, navegação 100% teclado, HTML semântico + ARIA correto, alvos de toque ≥ 44×44px, estrelas acessíveis via sr-only, prefers-reduced-motion respeitado e fontes nunca abaixo de 14px (body 16px mínimo).

CRIT: 10px breadcrumb → 14px+ CRIT: 12px body → 16px CRIT: sem CTA no hero WARN: sem foco visível → outline 3px WARN: estrelas emoji → sr-only WARN: breadcrumb sem ARIA → corrigido OK: alvos ≥ 44px em todos OK: prefers-reduced-motion

Ranking Final — Melhor para Pior em Acessibilidade

  1. 1
    Proposta B — Sidebar Vertical
    Separação semântica mais clara entre navegação, conteúdo e metadados via landmark regions. O sidebar garante hierarquia visual sem depender de cor. Contraste excelente em todos os pares. Meta strip acessível como tabela de dados implícita. Melhor implementação de skip-link e focus order.
  2. 2
    Proposta E — Magazine Editorial
    H1 de impacto tipográfico legível (4rem), kicker com tag visual + contexto textual. Meta strip como dl/dd implícita com foco funcional. CTA duplo com hierarquia clara. Pequeno custo: layout mais complexo exige mais teste de responsividade.
  3. 3
    Proposta A — Evolução Conservadora
    Correção fiel ao layout atual. Breadcrumb com chevron acessível, meta grid com bordas, CTA presente. Perde para B e E por ter menos diferenciação visual entre regiões, mas é a migração de risco mais baixo para a equipe.
  4. 4
    Proposta C — Coluna Terracota Invertida
    Contraste excelente (branco sobre terracota ≥ 4.8:1). Risco: usuários com daltonismo vermelho-verde podem ter dificuldade com a coluna esquerda. Mitigação parcial via peso de fonte, mas precisa de teste real com usuários.
  5. 5
    Proposta D — Full-Bleed Escuro
    Esteticamente a mais ousada e com bom contraste (texto sobre escuro ≥ 7:1). Porém, o depoimento em itálico fino sobre fundo semitransparente pode falhar em monitores com menos de 100% de calibração. Requer mais QA de contraste real. Complexidade do layout de imagens aumenta risco em mobile.

Evolução conservadora · Grid 2-col

Proposta A — Layout Original Corrigido

Problema resolvido: Mantém a estrutura conhecida pela equipe (flex 2-col) mas corrige as falhas WCAG críticas. Breadcrumb ganha aria-label e aria-current="page". Estrelas tornam-se SVG com aria-label descritivo. H1 sobe para 28px mínimo, body para 16px, breadcrumb para 14px. Meta grid separada da coluna branca por borda neutra. CTA de agendamento entra no hero acima do fold. Trade-off: menos impacto visual; identidade fotográfica precisa vir inteiramente das imagens — a coluna esquerda permanece branca sem âncora de cor.

Proposta B — Sidebar Vertical com Meta Strip

Problema resolvido: Adiciona um sidebar de 80px com cor terracota que ancora a identidade do ensaio sem depender só das imagens. C
Evolução segura · Coluna terracota invertida

Proposta C — Coluna de Identidade Terracota

Problema resolvido: A coluna esquerda branca "sem personalidade" é substituída por fundo terracota — a cor do ensaio viaja do accent para toda a área de texto, criando identidade imediata. Branco sobre terracota garante contraste de 4.8:1. CTA invertido (branco com texto terracota escuro). Meta strip na coluna direita, abaixo das imagens, sem conflito visual. Trade-off: Usuários com protanopia ou deuteranopia (deficiência vermelho-verde) podem perceber a coluna esquerda como marrom-acinzentada — o conteúdo permanece legível, mas o impacto da cor se perde. Recomendado teste com usuários cegos para cor antes de produção.

Proposta D — Hero Escuro com Imagem Atmosférica

Problema resolvido: Inverte o paradigma — imagem ocupa a metade esquerda em full-bleed sem padding, texto branco sobre quase-preto garante contraste AAA. O segundo frame da imagem "flutua" sobre o primeiro como polaroid com borda escura, criando profundidade. Meta strip discreta na base do painel direito. O foco outline: 3px solid #FFF é altamente visível sobre fundo escuro. Trade-off: Depoimento em itálico sobre fundo semitransparente precisa de QA cuidadoso — a opacity 0.8 do texto pode cair abaixo de 4.5:1 em alguns monitores. Complexidade do layout de imagem sobreposta aumenta risco em mobile. Requer mais QA cross-browser.

Proposta E — Tratamento Magazine Editorial

Problema resolvido: H1 de 4rem (64px) torna o nome da cliente o elemento dominante da página — hierarquia visual inequívoca. O kicker com tag colorida + ícone de localização separa categoria/contexto do título sem depender de tamanho relativo. Meta strip na base como dl implícita com três colunas de dado + depoimento integrado como quarta coluna. O design trata o nome da cliente como manchete, elevando a percepção de luxo da fotografia. Trade-off: H1 de 4rem precisa de line-height muito apertado (1.0) — garantir que não haja problemas de leitura em usuários com dislexia que aumentam texto no browser.

Notas de Implementação WCAG 2.2 AA

Critérios atendidos

  • ✅ 1.1.1 Non-text Content — imagens com alt descritivo ou aria-label no wrapper role="img"
  • ✅ 1.4.3 Contrast (Min) — todos os pares calculados ≥ 4.5:1
  • ✅ 1.4.4 Resize Text — fontes em rem/clamp, sem px fixo em texto
  • ✅ 1.4.12 Text Spacing — sem overflow ao aumentar line-height 1.5×
  • ✅ 2.1.1 Keyboard — toda interação acessível via Tab/Enter
  • ✅ 2.4.3 Focus Order — DOM order = visual order em todas as propostas
  • ✅ 2.4.7 Focus Visible — outline 3px sólido em todos os focusáveis
  • ✅ 2.4.11 Focus Not Obscured — foco nunca coberto por sticky elements
  • ✅ 2.5.3 Label in Name — aria-label contém texto visível do botão
  • ✅ 2.5.8 Target Size (AA) — mínimo 44×44px em links/botões
  • ✅ 3.1.1 Language of Page — lang="pt-BR" no html
  • ✅ 4.1.2 Name Role Value — landmarks, roles, labels ARIA corretos

Padrões ARIA usados

  • <nav aria-label="Caminho de navegação"> — breadcrumb landmark
  • aria-current="page" — item ativo no breadcrumb
  • <article aria-labelledby="id"> — hero como região nomeada pelo H1
  • role="img" aria-label="..." — wrapper de imagens placeholder
  • aria-hidden="true" — SVG decorativos, sidebar visual
  • <blockquote cite="..."> — depoimento com fonte citada
  • aria-label="Avaliação X de 5 estrelas" — estrelas acessíveis
  • role="list" + role="listitem" — meta grid como lista semântica
  • <a href="#main-content"> — skip link funcional para bypass
  • prefers-reduced-motion — todos os transitions removidos
  • rel="noopener noreferrer" — links externos seguros
  • <dl><dt><dd> — meta dados como lista de definição (Prop. E)
