B

Overlay Informacional sobre Imagem Plena

Evolução Segura

Problema resolvido: imagens dominam o espaço (o produto é fotografia), enquanto o conteúdo informacional vive num overlay com gradiente sólido — sem perda de legibilidade. Meta em grid 2×2 resolve fragmentação. Depoimento flutua no canto inferior direito como prova social sem competir com o título. Trade-off: se as fotos tiverem qualidade inconsistente ou tons muito claros à esquerda, o gradiente pode não proteger o texto. Resolve com overlay fallback de opacidade fixa.

C

Split Invertido — Imagem Dominante à Esquerda

Evolução Segura

Problema resolvido: CTA acima do fold via bloco meta+CTA fixo no bottom do col-text; segunda foto como thumbnail superposta resolve o problema de mostrar 2 fotos sem dividir igualmente — a composição fica mais editorial. Badge de contagem de fotos dá hint da galeria. Trade-off: thumbnail superposto pode colidir com UI em mobile se não tratado; solução: display:none abaixo de 600px. Requer fotos com boa composição no 1/3 superior.

D

Editorial Fotográfico — Imersão Total

Proposta Ousada

Problema resolvido: o site deve sentir como fotografia, não como CMS. Aqui as imagens são o hero — full-bleed, mosaico assimétrico. O texto vive sobre tint escuro com tipografia grande e respira. Mostarda usada apenas no eyebrow (segundo look da Camila), terracota no CTA, tudo mais quieto. Trade-off: o mais impactante, mas o mais dependente de imagens de alta qualidade técnica e composição forte. Não degrada bem com fotos medianas. Também é a maior mudança de código em relação ao template existente.

Formato Revista — 3 Colunas Editoriais

Proposta Ousada

Problema resolvido: a galeria abaixo do fold sem hint visual — aqui a barra "Ver 32 fotos" é parte integrante do hero, imediatamente abaixo. O layout de revista em 3 colunas separa editorialmente: texto narrativo / imagens / informação+conversão. O título em banda terracota full-width é o tese visual único desta proposta. Trade-off: mais complexo para implementar em Nuxt; grid de 3 colunas colapsa para 1 coluna mobile com mais scrolling necessário. É o layout que mais diverge do código atual.

Ranking das 5 Propostas

1

Proposta C — Split Invertido

Melhor equilíbrio entre impacto visual (imagem maior, à esquerda, como produto principal), robustez de implementação e CTA acima do fold. O thumbnail superposto resolve a "segunda foto" de forma editorial sem dividir o espaço visualmente. Meta e CTA agrupados no rodapé do col-text criam uma progressão narrativa clara: veja → leia → confie (depoimento) → aja. Degrada bem no mobile.

2

Proposta A — Evolução Estrutural

Menor fricção de implementação — é o layout atual corrigido, não substituído. Corrige todos os problemas críticos de produto: tipografia ilegível, CLS por aspect-ratio faltante, skeleton loading, estados de erro. Ideal se o objetivo for uma sprint rápida sem risco de regressão visual ou de aprovação.

3

Proposta D — Editorial Fotográfico

Visualmente o mais impactante e o mais coerente com a identidade de uma fotógrafa autoral. O risco é a dependência de fotos com qualidade técnica consistente — em imagens medianas o tint escuro mata a atmosfera ao invés de realçá-la. Recomendado para lançamento oficial com curadoria das fotos do hero.

4

Proposta E — Revista 3 Colunas

A solução mais completa para o problema do "gallery hint" abaixo do fold, e a mais editorial. O título em banda terracota full-width é um statement forte. O custo é a complexidade de implementação — 3 colunas com lógica de colapso mobile é mais complexo que o grid atual. Indicado para uma versão 2.0 do site.

5

Proposta B — Overlay Informacional

Conceitualmente válida (imagens dominam o espaço) mas o overlay com gradiente é a solução mais frágil: a legibilidade do texto depende da composição específica de cada foto. Em ensaios com fundos claros à esquerda (como os fundos bege-nude da Camila), o gradiente não protege suficientemente o texto. Exigiria um fallback de opacidade dinâmica por ensaio.
