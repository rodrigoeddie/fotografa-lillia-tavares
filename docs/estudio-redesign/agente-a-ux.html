Ranking das 5 Propostas — Melhor → Pior

1

P3 — Guiada (Progressive Disclosure + Sticky CTA)

Melhor equilíbrio entre UX e conversão. O accordion de cenários reduz sobrecarga cognitiva sem esconder informação. O sticky CTA contextual mantém a ação visível sem ser agressivo. Nav com âncoras de seção resolve a orientação sem alterar identidade visual. É defensável para a cliente e implementável no Nuxt sem risco.

UX 9.5 · Risco 3
2

P2 — Focada (Menos Ruído, Mais Conversão)

Stats bar + prova social no hero resolvem diretamente a objeção "não sei se é para mim". Cards com filtros dão exploração sem mosaico ambíguo. Nav com 3 pills reduz paralisia de escolha. Simples de implementar e fácil de A/B testar. Perde para P3 por não ter indicadores de progresso contextual.

UX 8.8 · Risco 2
3

P5 — Radical (Configurador de Ensaio)

Foot-in-the-door funciona bem para públicos indecisos — cada clique cria micro-comprometimento. O preview reativo é diferenciador forte no mercado fotográfico local. Cai para 3º porque requer mais JS confiável, pode intimidar usuários mais velhos, e o ganho depende de dados que ainda não existem. Recomendo testar como LP separada primeiro.

UX 8.2 · Risco 6
4

P1 — Refinada (Evolução Segura)

Resolve todos os problemas listados no brief (hierarquia, breadcrumb, nav ativa, CTA com peso) sem surpresas. A proposta mais segura para uma cliente conservadora. Perde para P2 e P3 porque não adiciona diferenciação — corrige o que estava errado mas não melhora o que estava apenas médio.

UX 7.8 · Risco 1
5

P4 — Imersiva (Full-Viewport + Scroll Horizontal)

A experiência visual é genuinamente diferenciada e compite com Instagram. Porém hero full-viewport atrasa o acesso à informação de conversão, scroll horizontal não é intuitivo em desktop, e o floating CTA com pulso pode irritar no uso prolongado. No mundo real funcionaria melhor com scroll behavior real — como protótipo standalone é a menos convincente.

UX 7.2 · Risco 7
