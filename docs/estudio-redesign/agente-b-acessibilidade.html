Proposta 3 — Estrutura de Conteúdo Clara

Resolve a hierarquia de headings completamente: apenas um H1 (página), H2 para seções, H3 para subseções. Adiciona landmark regions explícitas (header/nav/main/aside/footer), tabindex gerenciado corretamente no carrossel, e live region para anúncio de slides. Mosaico vira uma galeria semântica com figcaption em cada foto. A seção de fundos infinitos usa uma tabela de comparação acessível para descrever os cenários. Trade-off: layout levemente mais estruturado/editorial que o original orgânico.

Proposta 4 — Alto Contraste + Tipografia Ampliada (Extrapola)

Proposta que vai além da correção mínima: usa apenas preto (#1a1917, 18:1) e branco para garantir contraste máximo, eleva tamanhos de fonte (body 18px, H1 2.5rem) e garante espaçamento entre linhas de 1.8 para leitura confortável. Remove elementos puramente decorativos que confundem leitores de tela. CTA com ícone SVG inline com aria-hidden, texto sempre visível. Skip links múltiplos (conteúdo, navegação, rodapé). Modo alto contraste do sistema respeitado. Trade-off: afasta mais da identidade visual original; paleta quase monocromática.

Proposta 5 — Layout de Alto Impacto com Acessibilidade Total (Extrapola)

Proposta mais ousada: Nav horizontal com menu de hambúrguer acessível (aria-expanded, foco trapeado no mobile), hero full-width com sobreposição de texto garantindo contraste mínimo por overlay escuro, carrossel com autoplay desabilitável pelo usuário (WCAG 2.2 critério 2.2.2), e animações completamente condicionais ao prefers-reduced-motion. Toda cor de texto sobre foto usa garantia de contraste via sombra de texto + overlay. Seção de fundos com comparador interativo teclado-operável. Trade-off: mais JS, mais complexidade — mas cada interação tem fallback sem JS acessível.

Autoplay controlável ✓ Menu hambúrguer acessível ✓ Foco trapado no menu ✓ Contraste sobre foto ✓ Fallback sem JS ✓ GSAP reduced-motion ✓ Contraste ≥4.5:1 ✓

Correções críticas que qualquer proposta deve ter (WCAG 2.2 AA obrigatório)

  1. #7B785B sobre #F4F2E9 = 3.0:1 — FALHA. Escurecer para ≥#5a5740 ou mudar bg para #fff.
  2. #BD9187 (rose) sobre branco = 2.8:1 — FALHA. Nunca usar rose como cor de texto; apenas decorativo.
  3. Múltiplos H1 — permitido somente 1 por página (exceto em seções com landmark separado).
  4. Swiper sem aria-label, live-region e tabindex gerenciado — cada slide iativo deve ter tabindex="-1" em links/botões internos.
  5. Skip to content link — ausente no site atual; obrigatório para usuários de teclado.
  6. Mapa estático sem alt — toda imagem informativa precisa de alt descritivo ou endereço textual equivalente.
  7. Footer ícones sem label — ícone SVG/emoji sem texto visível precisa de aria-label no elemento <a>.
  8. prefers-reduced-motion — GSAP e ScrollTrigger devem verificar window.matchMedia('(prefers-reduced-motion)') antes de registrar tweens.
